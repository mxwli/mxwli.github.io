Fast Fourier Transform Reference (non recursive)

The Cooley-Tukey Algorithm is an algorithm made to compute the Discrete Fourier Transform in O(N log N) time. Since the algorithm is most commonly expressed in terms of a recursive relation, most implementations of it end up being recursive. The problem with this approach is that recursive implementations tend to be slower than iterative implementations.

The code linked below implements the algorithm iteratively.

VectorGE

In highschool I set out to build a game on top of a 2D engine idea I had. Essentially, every real element in the game would be represented by a set of 2D vectors. Map walls were represented by 2 vectors, while entities were assigned a vector and a radius.

All the functionalities of this game were built upon these two rules alone. The source code had gotten really messy over time, and I wanted to port it to a web page, so I'm rewriting it in C with raylib. The rewrite is currently a work in progress

Procedurally Generated Bullet Hell

For the final project of my High School computer science class, I had the idea of procedurally generating a bullet hell game using certain rhythm elements of a song. To obtain these rhythm elements, I self taught the Cooley-Tukey FFT algorithm, which converts a signal into frequency information about the signal. Essentially, I was picking out the individual "notes" of the audio
